> **V1.1 update.** After shipping V1, we realised `[record]` and
> `[causal]` were a category mistake — brackets are intuitively *tags*
> (transparent, removable), but those two are *effectful*. V1.1 inverts
> them: `[record]` is now a `--record` CLI flag, `[causal]` is the
> default for every memory slot. The brackets that remain are the ones
> that genuinely behave like tags. See `docs/SEMANTICS.md §6` for the
> normative discussion. None of the 100 use cases or 10 CLBG challenges
> needed to change — only the V1 example files I added.

## How to try v1

```sh
# Clone, build
git clone https://github.com/anthropics/soma   # (or wherever you host)
cd soma/compiler && cargo build --release
SOMA=./target/release/soma

# Session types
$SOMA check ../examples/v1/01_session_types_auction.cell
# Try removing one of the handlers — the compiler will refuse to deploy.

# Record / replay (V1.1: --record is a CLI flag, not a handler annotation)
$SOMA run --record ../examples/v1/02_replay_trader.cell
$SOMA replay       ../examples/v1/02_replay_trader.cell
# Uncomment the now_ms() line — replay will report the divergence
# and tell you exactly how to make the handler deterministic.

# Lean 4 proof export
$SOMA verify ../examples/v1/03_prove_payment.cell --export-proof
cat ../proofs/payment.lean        # ready for `lake build`

# Adversary-quantified verification
$SOMA verify ../examples/v1/04_adversary_quorum.cell
# Every PASS is now stamped (under network ∧ llm).

# Causal memory
$SOMA run ../examples/v1/05_causal_orders.cell
# Every write extends the slot's vector clock; clock_of() and
# happens_before() expose them to handler code.
```

## What it adds to the language (line count)

  - **lexer**: 3 new keywords
  - **AST**:   4 new section variants + supporting structs
  - **parser**: 3 new top-level parsers, ~180 lines
  - **checker**: 2 new modules (`protocols.rs`, `adversary.rs`), ~120 lines
  - **codegen**: 1 new emitter (`lean4.rs`), ~110 lines
  - **runtime**: 1 new backend (`CausalBackend`), ~80 lines
  - **interpreter**: `record_log.rs` (~150 lines), nondet detection (~10 lines)
  - **commands**: `replay.rs` (~140 lines)
  - **stdlib**:   `causal` (no-op compat) + `uncausal` opt-out
  - **CLI**:      `Replay` command, `--export-proof` flag, `--record` flag

Total: roughly **1 200 lines added** in V1, **~50 lines net change** in
V1.1 (mostly removals + the wrapper). None of the 100 verified use
cases or 10 CLBG challenges needed to change.

## V1.1 inversion: brackets are compiler output, not user input

After V1 shipped, we recognised that `[record]` and `[causal]` were a
category mistake — brackets are intuitively *tags*, but those were
*effectful*. V1.1 inverts them:

  | V1 syntax                                | V1.1 syntax                                  |
  |------------------------------------------|----------------------------------------------|
  | `on tick(p) [record] { … }`              | `on tick(p) { … }` + `soma run --record`     |
  | `orders: Map [causal]`                   | `orders: Map`  *(causal is the default)*     |
  | `[ephemeral]` slot bracket required      | bracket optional, `orders: Map` is legal     |

The brackets that remain are the ones that genuinely behave like tags:
`[native]` (codegen flag), `[persistent]` (storage backend), `[consistent]`
(memory model). The effectful ones moved out.

This subtraction does not break the 110 challenges. Only the V1 example
files I added needed to change.

## What was punted to v1.2

Tracked in `docs/SEMANTICS.md` §6:

  - session-type *ordering* (V1 checks exhaustiveness only)
  - static causal happens-before (V1.1 has runtime clocks, not types)
  - adversary-aware verification (V1 stamps but doesn't reweight)
  - backend-equivalence theorem (V1 has differential tests)
  - Lean kernel discharge (V1 emits theorem skeletons)
  - explicit `uses P as role` binding (V1 matches role to cell name)
  - `[pure]` / `[persistent]` / `[consistent]` inference (continue subtracting brackets)
