# Soma v1 — what landed

Status of `TARGET_SOMA_V1.md`. This document is the changelog; the
normative semantics live in `docs/SEMANTICS.md`, and the runnable
demos live in `examples/v1/`.

| #  | Feature                                  | Shipped? | Demo                                  |

All five demos pass `soma check`, run end-to-end with `soma run`, and
exercise the V1 features they advertise. The semantics doc is
normative — the implementation tracks it; deviations are bugs.

## How to try v1

```sh
# Clone, build
git clone https://github.com/anthropics/soma   # (or wherever you host)
cd soma/compiler && cargo build --release
SOMA=./target/release/soma

# Session types
$SOMA check ../examples/v1/01_session_types_auction.cell
# Try removing one of the handlers — the compiler will refuse to deploy.

# Record / replay
$SOMA run    ../examples/v1/02_replay_trader.cell
$SOMA replay ../examples/v1/02_replay_trader.cell
# Uncomment the now_ms() line — replay will report the divergence
# and tell you exactly how to make the handler deterministic.

# Lean 4 proof export
$SOMA verify ../examples/v1/03_prove_payment.cell --export-proof
cat ../proofs/payment.lean        # ready for `lake build`

# Adversary-quantified verification
$SOMA verify ../examples/v1/04_adversary_quorum.cell
# Every PASS is now stamped (under network ∧ llm).

# Causal memory
$SOMA run ../examples/v1/05_causal_orders.cell
# Every write extends the slot's vector clock; clock_of() and
# happens_before() expose them to handler code.
```

## What it adds to the language (line count)

  - **lexer**: 3 new keywords
  - **AST**:   4 new section variants + supporting structs
  - **parser**: 3 new top-level parsers, ~180 lines
  - **checker**: 2 new modules (`protocols.rs`, `adversary.rs`), ~120 lines
  - **codegen**: 1 new emitter (`lean4.rs`), ~110 lines
  - **runtime**: 1 new backend (`CausalBackend`), ~80 lines
  - **interpreter**: `record_log.rs` (~150 lines), nondet detection (~10 lines)
  - **commands**: `replay.rs` (~140 lines)
  - **stdlib**:   `[causal]` property declaration (~15 lines)
  - **CLI**:      `Replay` command, `--export-proof` flag

Total: roughly **1 200 lines added**, all behind explicit annotations.
None of the existing examples needed to change. Five features, one
rigor doc, zero migrations.

## What was punted to v1.1

Tracked in `docs/SEMANTICS.md` §6:

  - session-type *ordering* (V1 checks exhaustiveness only)
  - static causal happens-before (V1 has runtime clocks, not types)
  - adversary-aware verification (V1 stamps but doesn't reweight)
  - backend-equivalence theorem (V1 has differential tests)
  - Lean kernel discharge (V1 emits theorem skeletons)
  - explicit `uses P as role` binding (V1 matches role to cell name)
