# Soma v1 ‚Äî what actually landed

> **Subtraction note.** V1 originally proposed five features
> (`protocol`, `prove`, `adversary`, `causal`, `[record]`). After
> shipping them, we audited each against a single test:
> *does it bring value over what already existed?*
>
> Four failed the test:
>
>  - **`protocol`** only checked exhaustiveness, which is exactly what
>    `face { signal X(args) }` plus the existing handler check already
>    did. Without ordering checks (deferred to V1.2), it was face
>    contracts in fancier syntax.
>  - **`prove` ‚Üí Lean 4** emitted theorem stubs whose proof body was
>    `trivial`. Nobody could `lake build` it into a real proof. It
>    was an aspirational stub pretending to be proof export.
>  - **`adversary`** stamped verifier output with `(under network ‚àß llm)`
>    *without the verifier actually modeling those failures*. This was
>    actively misleading: a "‚úì no deadlocks (under network ‚àß llm)"
>    message implied robustness against network drops, but the
>    verifier hadn't modeled drops at all.
>  - **`causal` memory** in single-replica mode is just a sequence
>    number wearing vector-clock clothes. Without distribution
>    (deferred to V1.2), `happens_before` is total ordering with extra
>    ceremony.
>
> One passed:
>
>  - **`soma run --record` + `soma replay`** ‚Äî records every handler
>    invocation to a `.somalog` file, replays deterministically against
>    a fresh interpreter, detects nondeterminism via `now()`/`random()`
>    interception, reports each divergence with a suggested fix.
>    Works end-to-end. Solves a real problem.
>
> V1.2 subtracted the four that didn't carry their weight. The git log
> records the full arc ‚Äî V1, V1.1 inversion, V1.2 subtraction ‚Äî so the
> design history is honest. None of the 100 verified use cases or 10
> CLBG challenges needed to change at any step.

## What ships in V1

| Feature                       | Status                                          |
|-------------------------------|-------------------------------------------------|
| `soma run --record`           | recording mode, opt-in CLI flag                 |
| `soma replay <file>`          | deterministic re-execution + divergence reports |
| nondeterminism detection      | `now`, `random`, etc. flagged with fix hints    |
| Honest semantics doc          | `docs/SEMANTICS.md` (trimmed)                   |
| One demo                      | `examples/v1/02_replay_trader.cell`             |

## How to try it

```sh
cd compiler && cargo build --release
SOMA=./target/release/soma

# Record a run
$SOMA run --record ../examples/v1/02_replay_trader.cell
# Writes ../examples/v1/02_replay_trader.somalog

# Replay it
$SOMA replay ../examples/v1/02_replay_trader.cell
# All entries should re-execute with no divergence.

# Now uncomment the now_ms() line in the .cell and re-run with --record,
# then re-run replay. You'll see something like:
#
#   divergence at entry #1: trader.tick
#       args:     100
#       recorded: 102
#       replayed: 103
#       cause:    nondeterminism in handler ‚Äî calls to now_ms
#       fix:      mark the handler [pure] or pass the clock as an explicit input parameter
```

## What's punted to V1.2 (with intent to actually deliver)

  - **session-type ordering**, not just exhaustiveness. The current
    `face { signal X }` check already covers exhaustiveness; what's
    missing is "receiver may not handle Msg‚ÇÇ before Msg‚ÇÅ if the
    protocol orders them." That's the part that justifies a new
    construct.
  - **adversary-aware verification.** Stop stamping; actually model
    drops/reorders/partitions in the state-space exploration.
  - **distributed causal memory.** Vector clocks become useful when
    there are multiple replicas to compare.
  - **proof-carrying export with discharged kernels.** A real Lean 4
    pipeline that produces `lake build`-clean files.

Each of these will only ship when it brings value over what already
exists. The principle from V1.2:

> **Ship subtraction. Ship features that work end-to-end. Don't ship
> annotations the compiler can derive, and don't ship "hooks" for
> features that aren't actually doing anything yet.**

## The single sentence

> Soma v1 is the language where every production incident is a single
> command away from being replayed deterministically on your laptop.
